Introduction

Git is a version control system used to record changes to a file over time so that you can recall specific versions later on. It allows someone to save different versions of a web design or image so that they basically don't have to back track and delete and progress. You can use it to compare changes over time or to see who modified a file and when they did so.

The goal of this page is to get you started with git and teach you the following:

  • How to install git on your computer
  • How to interact with git using the command line
  • How to set up a repository
  • How to push your code to your repository
  • Where to find more information about Git
The History of Git

Git was created by the same team that created the Linux Kernal. They originally changed the software via passing around patches and archiving files but in 2002, they decided to start using a proprietary DVCS called bit keeper.

In 2005, the relationship between the two comanies began to crumble and BitKeeper decided to remove their products free od charge status for Linux. This pushed Linux to start creating their own version control system called Git and thus the system was born.

Installing Git

Installing Git is quick and easy. Depending on your machine ,you may even be able to do it all from the command line. If you are using a linux machine, you will need to open your terminal and type:

$ sudo dnf install git-all

If you are using a Debian based distribution such as Ubuntu type:

$ sudo apt install git-all

If you are using a Windows or a Mac device, go to this site. After the download is complete, you will have access to all of the tools that git provides.

Getting Started

One of the first things you can do after installing git is update it with some of you information; to let the system know who you are. This is important because every commit (more on this later) will use this information to keep track of who is updating the repository.

$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com

If you would like to set up your environment to use a different editor you can use the command:

$ git config --global core.editor emacs

Where emacs can be replaced by any editor of your choice

Setting Up a Repository

Turning a directory into a repository can be achieved either by:

  • Taking a local directory and turning it into a repository.
  • Cloning and existing repository

To create a new repository from a local directory simply navigate to the folder using the command line and type:

git init
git add *.c
git add LICENSE
git commit -m 'initial project version'

This will set up your repository to start tracking files. If you want to clone an existing repo, you can use the clone command in the following way:

git clone https://github.com/ex-user/ex-repo

This example uses github as a source for the repo; it is highly recommended that you set up an account of your own at github.com

Pushing

Once you have your repository set up you can begin staging your code and tracking it. using the git status will tell you what files have been tracked. To begin tracking files you need to use the add function:

$ git add "file_name"

where "file_name" is the file you would like to be tracked. You can replace this with an astrisk (*) if you would like all of your files to be tracked.
Next step is to use the commit function to add a message with your files like so:

$ git commit -m "initial commit"

The added files will be commited with the message that you specify between the quotation marks. The -m tells the system that you want to add a message on the command line, otherwise you will be taken to your text editor of choice.

The final step to staging your code to the repository is to use the push function like so:

$ git push -u origin master

The -u origin master part is important if this is your first push for the repository but can be ignored in the future.

And that's it! Those are the basics of Git and will allow you to upload your code to a decentrilized version control system and help make it easier to collaborate on a piece with other people.
